**Success for May 2026 means shipping a stable `v1.2.0` Git tag on `master` that contains:**

1. **Core Epic C operational proof** — RC gates signed, evidence archived ([Epic C bundle](./evidence/v1.2.0-epic-c/)), including staging **`AGG_KEY_VERSION`** cutover log (**gate C-2**, release blocker).
2. **Day 60 verification infrastructure** — reproducible vertical benchmark harness (`seed: 42`), challenge-policy escalation path, ingress scorecard MVP, and CI/docs that prove them on the RC SHA ([Day 60 sprint tracker](./v1.2.0-day60-sprint-tracker.md)).

**Success for May 2026 does NOT mean:**

- Closing **30** open [12-month roadmap](https://github.com/pamu512/tarka/blob/master/docs/docs/guides/tarka-12-month-roadmap-execution-kit.md) umbrella epics ([#127](https://github.com/pamu512/tarka/issues/127)–[#156](https://github.com/pamu512/tarka/issues/156), [#55](https://github.com/pamu512/tarka/issues/55)).
- Merging **`1.3.0-beta`**, **[PR #170](https://github.com/pamu512/tarka/pull/170)**, or local analyst UI batches into `master` before the tag.
- Delivering Trust Center, signed release artifacts, compliance evidence auto-packs, or unified analyst workbench (all **June / v1.3.0**).

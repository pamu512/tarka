# Release v1.3.0 (Planned)

Planned release date: `2026-06-29`

## Theme

Enterprise trust center and release governance hardening.

## Journey coverage (J1–J5)

How this release advances [user journeys](../guides/journey-ship-bundles.md) (see that guide for full issue/release mapping):

| Journey | Role | Planned emphasis in v1.3.0 |
|--------|------|------------------------------|
| **J1 — Integrate** | Integrations / platform | Indirect: Trust Center and evidence exports should reference **deployment profiles** and integration posture where controls apply. |
| **J2 — Decide** | Risk / automation | Evidence exports include decision/policy lineage suitable for audit; promotion and typology version metadata where contracts allow. |
| **J3 — Investigate** | Analyst | Evidence export APIs and bundle-shaped outputs support investigation narratives and procurement evidence (feeds **#50** / agent alignment). |
| **J4 — Govern** | Compliance / leadership | **Primary:** Trust Center UI, evidence export APIs, CI release-governance checklist, signed artifacts—procurement-ready posture. |
| **J5 — Operate** | SRE / platform | Release gates, signed builds, and operational readiness views in Trust Center; runbook and health linkage as acceptance criteria. |

## Planned Highlights

- Trust Center UI for control posture and operational readiness.
- Evidence export APIs for audit and control validation.
- Release-governance checklist integrated with CI gates.
- Signed release artifact generation and publication workflow.

## Planned Breaking Changes

- None expected.

## Planned Validation

- Evidence exports verified for completeness and integrity.
- Trust Center UI acceptance tests for health/control views.
- CI release gate tests pass for governance checklist enforcement.

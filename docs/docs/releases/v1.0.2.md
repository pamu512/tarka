# Release v1.0.2 (patch)

Patch release: CI/security ergonomics and manual workflow reruns.

## Changes

- **CodeQL Advanced** and **Security scan** workflows declare **`workflow_dispatch: {}`** so GitHub registers manual runs: use **Run workflow** in the Actions tab or `gh workflow run "CodeQL Advanced" --ref master` / `gh workflow run "Security scan" --ref master` (ref must match your default branch).
- **CodeQL (Swift / iOS SPM):** build order and `set -e`-safe helpers; **`CODEQL_ACTION_FILE_COVERAGE_ON_PRS=true`** to keep PR file-coverage behavior when GitHub defaults change (April 2026+ notice).
- **Security scan:** Trivy SARIF uploads use **`github/codeql-action/upload-sarif@v4`** (already on trunk before this tag where applicable).
- **Docker images:** base **`apt-get upgrade` / `apk upgrade`** steps for security fixes (see recent `ci(security)` commits).
- **Benchmark smoke:** **`drift_score_smoke.py`** job for heuristic drift separation (see `scripts/benchmarks/`).

## Troubleshooting (CodeQL rerun)

If **`gh workflow run codeql.yml`** returns **422** (“disabled workflow”): the workflow file was turned off in the Actions UI. Re-enable it:

```bash
gh workflow enable codeql.yml
```

If **`gh workflow run "CodeQL Advanced"`** targets the wrong workflow ID (duplicate names): prefer **`gh workflow run codeql.yml --ref master`**.

Stale workflows pointing at removed files (e.g. a former `advancedcodeql.yml`) can be **disabled** in **Actions → … → Disable workflow** to avoid duplicate CodeQL runs alongside **`dynamic/github-code-scanning/codeql`** (default setup).

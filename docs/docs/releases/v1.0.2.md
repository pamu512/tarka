# Release v1.0.2 (patch)

Patch release: CI/security ergonomics and manual workflow reruns.

## Changes

- **CodeQL Advanced** and **Security scan** workflows accept **`workflow_dispatch`**, so maintainers can **Run workflow** from the Actions tab without an empty commit (fixes the prior “cannot rerun CodeQL via `gh workflow run`” limitation).
- **CodeQL (Swift / iOS SPM):** build order and `set -e`-safe helpers; **`CODEQL_ACTION_FILE_COVERAGE_ON_PRS=true`** to keep PR file-coverage behavior when GitHub defaults change (April 2026+ notice).
- **Security scan:** Trivy SARIF uploads use **`github/codeql-action/upload-sarif@v4`** (already on trunk before this tag where applicable).
- **Docker images:** base **`apt-get upgrade` / `apk upgrade`** steps for security fixes (see recent `ci(security)` commits).
- **Benchmark smoke:** **`drift_score_smoke.py`** job for heuristic drift separation (see `scripts/benchmarks/`).

## Version bumps

- Root Python meta package **`tarka`** `1.0.0` → **`1.0.2`** (`pyproject.toml`).
- Frontend **`tarka-ui`** **`1.0.2`** (`frontend/package.json` + lockfile root).
- **`tarka` CLI** installer state default version string aligned to **`1.0.2`**.

Service-level `pyproject.toml` versions are unchanged unless they already tracked `1.0.x` independently.

## GitHub issues

No open **`bug`**-labeled issues in the repo mapped 1:1 to this patch; roadmap **`enhancement`** items remain open by design.
